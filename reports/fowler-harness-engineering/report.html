Fowler《Harness Engineering for Coding Agent Users》来源报告

一句话结论：coding-agent 用户的外层 harness 是一个持续维护的控制系统：行动前用 feedforward 引导，行动后用 feedback 传感器纠偏，并把重复失败写回控制回路。

来源信息与阅读范围

  • 来源：Martin Fowler 网站，Birgitta Böckeler，2026-04-02。
  • 原始 URL：https://martinfowler.com/articles/exploring-gen-ai/harness-engineering.html
  • 当前规范 URL：https://martinfowler.com/articles/harness-engineering.html
  • 阅读范围：正文从 Contents 起的 14 个 H2 小节，重点核对控制方向、计算/推断形式、时机、调节类别和 harnessability。

问题与背景

文章从 coding-agent 用户的信任障碍出发：模型非确定、缺少本地语境，且自然语言自述不足以证明代码正确。作者不试图定义完整产品内部结构，而是研究用户能在 agent 周围增加哪些 guides 与 sensors，提高结果可信度并减少监督成本。

论证结构

文章先以 feedforward/feedback 建立控制方向，再用 computational/inferential 区分执行形式；随后把失败写回 steering loop，并按开发生命周期布置快速、昂贵和持续传感器；之后将控制目标分成可维护性、架构适应度和行为三类；最后讨论代码库本身的 harnessability、模板、人类角色与开放问题。

关键观点与证据

1. Harness 同时需要事前引导与事后纠偏

来源观点：feedforward guides 通过规则、参考资料和工具在行动前降低错误概率；feedback sensors 观察输出并让 agent 或人纠正。只使用一个方向会失去另一半控制能力。

[定位：全文 H2 #3（前馈与反馈），第 1 个列表及第 2 段]

2. 控制可以是计算式或推断式

来源观点：结构测试、脚本、codemod 和静态分析可以机械执行；AGENTS.md、skills 和 review agents 依赖模型解释。两种形式可以组合，例如 how-to 指令与 bootstrap 脚本共同约束启动流程。

[定位：全文 H2 #4（计算式与推断式控制），表格]

3. 重复失败应升级 harness

来源观点：人的工作不是每次单独纠正输出，而是在问题重复时改进 guides 与 sensors，让下一次失败更不可能发生。agent 也可以帮助生成结构测试、规则草案和自定义 lint。

[定位：全文 H2 #6（转向循环），第 1—2 段]

4. 传感器需要按成本和时效分层

来源观点：快速检查应尽量左移到提交前；昂贵审查可放在集成后；代码腐化、依赖与运行指标还需要持续传感器。harness 因而跨越一次变更和长期维护两个时间尺度。

[定位：全文 H2 #7（质量左移），第 1—5 段及两个列表]

5. 代码库的结构决定可 harness 程度

来源观点：强类型、明确模块边界和稳定 framework 自带可执行 affordances；绿地项目可以从第一天设计可治理性，而高技术债遗留系统往往最需要 harness 却最难建立。

[定位：全文 H2 #9（代码库可约束性），第 1 段；全文 H2 #10（环境基础条件），第 1—2 段]

核心概念与方法

  • Feedforward Guide：在生成前提供方向、规则和可用动作。
  • Feedback Sensor：在行动后观察偏差并触发自我纠正或人工审查。
  • Computational Control：由确定性程序执行的约束或检查。
  • Inferential Control：由模型解释规则或审查内容的控制。
  • Steering Loop：把重复问题转换为新的 guide 或 sensor。
  • Harnessability：环境结构允许建立有效控制的程度。

局限与适用边界

  • 文章有意聚焦 coding-agent 用户的“外层 harness”，不等同于产品运行时的完整构成。
  • inferential reviewer 仍可能与生成模型共享盲点；文章特别保留了对 AI 生成测试可信度的疑问。
  • 不同调节类别的可计算性差异很大，行为正确性通常比结构约束更难机械化。
  • 这是一套工程心智模型和实践议程，而非受控实验或成员资格测试。

[定位：全文 H2 #8（调节类别）中行为控制子节，第 10—12 段；全文 H2 #14（开放问题），第 1—3 段]

对 Agent Harness 主题的贡献

本文把 harness 从“组件集合”提升为控制论系统：信息怎样在行动前进入，结果怎样在行动后被感知，重复失败怎样改变下一版控制。它为主题中的 Control Mechanisms 和 Entropy Management 提供了最清楚的工程语言。
