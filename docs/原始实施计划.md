# PaperWiki：论文发现、精读与知识沉淀流程

## 总体方案

采用“现有 Skills 组合 + 轻量评分适配层”，形成可拆分、可组合的三段式流水线：

```text
研究方向 → 多源发现与质量评分 → 候选论文清单
论文链接/DOI/PDF → 单篇精读与可视化报告
阅读报告/个人笔记 → 人工确认 → 结构化知识沉淀 → 后续阅读建议
```

第一版采用 Markdown + Obsidian + Git，不引入向量数据库。三个阶段均可独立运行。

## Skills 与依赖

### 发现论文

- `openags/paper-search-mcp`：多源搜索与下载。
- arXiv + Semantic Scholar：CS/AI 默认检索与引用信息来源。
- NousResearch Hermes arXiv skill：种子论文、引用图和相关论文流程参考。
- DeerFlow systematic-literature-review：跨论文归纳的可选流程。
- `discover-papers`：统一规范化、去重、评分和输出。

评分权重为相关度 30%、venue 20%、引用影响 15%、时效 15%、代码/复现 10%、作者延续性 5%、新颖/热度 5%。输出总分、分项、证据覆盖率、理由，以及必读/推荐/候选/观察分档。缺失引用信息不得按零质量处理。

### 阅读论文

- `Wm78gh/paper-craft-skills`：深度解读、速览、漫画、导图和可视化报告。
- `read-paper`：接受 URL、DOI、arXiv ID、本地 PDF 或发现记录；下载和保留来源；输出报告及统一结构化元数据。

报告包含书目信息、研究问题、假设、方法、数据集、实验、结论、创新、局限、复现条件、公式、术语、关联论文，以及用户批注和阅读状态。

### 沉淀知识库

- InfraNodus `llm-wiki`：持久 Wiki 分层和增量工作流参考。
- `kepano/obsidian-skills`：Markdown、双链、Bases 和 Canvas 规范。
- `deposit-paper-knowledge`：将报告或现有笔记幂等写入 PaperWiki。

论文以 DOI 为主键，其次 arXiv ID，最后规范化标题哈希。自动生成内容与人工确认内容分离；更新不得覆盖用户批注。

## 完整流程

1. 输入方向、时间和数量，多源召回并去重。
2. 获取 venue、引用、作者、代码、日期、摘要，计算可解释综合分。
3. 用户选择论文，或直接提供论文而跳过发现。
4. 下载 PDF，记录来源、时间、版本和许可信息。
5. paper-craft 生成速览、深度报告和可视化内容。
6. 用户补充疑问、理解、评价和复现结果。
7. 校验元数据并写入论文页，增量更新概念、方法、数据集、主题、索引和日志。
8. 后续查询优先区分已读知识与外部新发现，并据知识缺口生成阅读建议。

状态为 `discovered → shortlisted → queued → reading → reviewed → deposited → revisited`，允许跳过前置状态。失败必须保留可恢复错误，不能静默跳过。

## 验收

- 真实检索近两年论文，验证多源去重、过滤、评分解释。
- 对经典论文和新论文验证评分不系统性偏旧。
- arXiv、DOI、出版社、PDF 最终收敛到相同主键。
- 完整处理论文并验证原文、报告、论文页、概念页和主题页链接。
- 重复沉淀不重复建页且不覆盖用户笔记。
- 第二篇相关论文增量更新共享概念和主题综述。
- 下载失败、限流、无 DOI、撤稿和冲突留下可恢复状态。
